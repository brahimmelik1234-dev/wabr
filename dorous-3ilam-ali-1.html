الإعلام الآلي، المعلومة و T.I.C

مفاهيم أساسية لفهم المادة
▼

الإعلام الآلي Informatique هو العلم الذي يهتم بمعالجة المعلومات بطريقة آلية باستعمال الحاسوب وبرامج مخصصة، بهدف حل مشكلات حقيقية في مجالات مختلفة (تعليم، طب، صناعة، إدارة…).

1. المعلومة (Information) والحاسوب

Information
المعلومة
هي كل ما يحمل معنى: رقم، كلمة، صورة، صوت، جدول، نتيجة تجربة… الحاسوب لا يفهم إلا المعلومات المكتوبة في شكل منظم يمكن معالجته.
Bit & Octet
وحدة قياس المعلومة
المستوي الأدنى هو البت Bit يأخذ القيمة 0 أو 1. ثمانية بتات تعطي Octet، ومنه الكيلو أوكتي، الميغا أوكتي… إلخ.
Ordinateur
الحاسوب
جهاز إلكتروني يستقبل المعطيات (دخول)، يعالجها وفق برنامج معيّن، ثم يعطي النتائج (خروج). يتكوّن من جزء مادي Matériel (الشاشة، لوحة المفاتيح، الذاكرة…) وجزء برمجي Logiciel (أنظمة التشغيل، البرامج، التطبيقات).

2. تكنولوجيا الإعلام والاتصال T.I.C

تكنولوجيا الإعلام والاتصال (TIC) هي استعمال الحواسيب، الشبكات والبرمجيات (مثل Google، البريد الإلكتروني، GPS، منصات الفيديو…) لتخزين المعلومات، معالجتها، نقلها ومشاركتها بسرعة كبيرة.

البريد الإلكتروني محركات البحث GPS منصات بث الفيديو التعليم عن بعد
💡
الفكرة الأساسية: الإعلام الآلي يهتم بمنطق حل المشكلات وتنظيم المعلومات، بينما T.I.C تهتم بالوسائل (الإنترنت، الشبكات، التطبيقات) التي تسمح باستعمال هذه الحلول في الحياة اليومية.

الخوارزميات والتفكير الخوارزمي

كيف نحول المشكلة إلى خطوات دقيقة؟
▼

1. تعريف الخوارزمية

الخوارزمية Algorithme هي سلسلة محددة ومنتهية من التعليمات البسيطة الواضحة، تُنفّذ بترتيب معيّن من أجل حل مشكلة أو إنجاز مهمة.

Problème → Algorithme
مراحل حل المسألة
لتحل مسألة بالحاسوب نمرّ غالبًا عبر المراحل التالية:
  • 1. فهم المشكلة وتحليلها (Décomposition).
  • 2. استخراج النمط والتشابه بين الأمثلة (Pattern recognition).
  • 3. اختيار ما هو مهم وإهمال التفاصيل الزائدة (Abstraction).
  • 4. كتابة خوارزمية عامة قابلة للتنفيذ (Algorithmic thinking).
Qualités
خصائص الخوارزمية الجيدة
  • غير غامضة (Unambiguous): كل تعليمة لها معنى واحد فقط.
  • حتمية (Déterministe): نفس المعطيات تعطي نفس النتيجة.
  • منتهية (Finie): تنتهي بعد عدد محدود من الخطوات.
  • ممكنة التنفيذ (Faisable): يمكن تنفيذها زمنيًا وعمليًا.
  • مستقلة عن لغة البرمجة (Indépendante).

المتغيرات، الثوابت وأنواع البيانات

كيف نخزن المعلومات داخل الخوارزمية؟
▼

1. المتغير، الثابت والمعرّف

Variable
المتغير
خانة في الذاكرة يمكن أن تتغير قيمتها أثناء تنفيذ الخوارزمية. كل متغير له اسم (معرّف)، نوع، وقيمة.
Constante
الثابت
قيمة ثابتة لا تتغير أثناء تنفيذ الخوارزمية (مثل π = 3.14 أو TVA = 0.19)، تُستعمل في الحسابات.
Identificateur
المعرّف
هو اسم المتغير أو الثابت. يجب أن يبدأ بحرف، بدون فراغات أو رموز خاصة، ويفضّل أن يكون معبّرًا عن المعنى (Somme، Moyenne، Rayon…).

2. أنواع البيانات الأساسية

النوع الاسم بالفرنسية مثال للقيم
عدد صحيح Entier -5 ، 0 ، 200
عدد حقيقي Réel 1.4 ، -254.55
محرف واحد Caractère 'm' ، 'A' ، '?'
سلسلة حروف Chaîne "Algorithme1" ، "info"
منطقي Booléen Vrai / Faux
📌
عند التصريح بالمتغيرات في الخوارزمية نكتب:
Var NomVariable : Type مثلًا: Var age : Entier ، Var Moyenne : Réel،
ونعطي الثوابت بالشكل: Const PI = 3.14.

1. التعليمات الأساسية

Affectation
إسناد قيمة لمتغير
نجعل متغيرًا يأخذ قيمة معينة أو نتيجة تعبير. نستعمل غالبًا السهم
<- أو ←.
x <- 5

قراءة المعطيات
تستعمل لطلب إدخال قيم من المستخدم.
Lire x
Lire x, y, z
Écrire
إظهار النتائج
تستعمل لعرض رسائل أو قيم المتغيرات.
Écrire "La somme =", som

التعليقات
تعليقات توضيحية لا تُنفّذ، تساعد في فهم الخوارزمية. نكتبها عادة بعد رمز خاص مثل //.

الصنف الرموز مثال

البنى الشرطية: Si…Alors…Sinon

اختبار الشروط واتخاذ القرار
▼

1. بنية بسيطة: Si … Alors … Fin Si

نستعملها عندما نريد تنفيذ تعليمات فقط إذا تحقق شرط معيّن.

Si condition Alors
  Instructions // تُنفّذ إذا كان الشرط Vrai
Fin Si

2. بنية مع فرع آخر: Si … Alors … Sinon … Fin Si

Si condition Alors
  Instructions_1 // حالة Vrai
Sinon
  Instructions_2 // حالة Faux
Fin Si

3. مثال: قيمة مطلقة لعدد

Algorithme ValeurAbsolue
Var n, nabsolue : Entier
Début
  Écrire "Entrer un nombre entier"
  Lire n
  Si n < 0 Alors
    nabsolue <- -n
  Sinon
    nabsolue <- n
  Fin Si
  Écrire "La valeur absolue est", nabsolue
Fin

البنى التكرارية: Tant que, Pour, Répéter

تكرار التعليمات عدة مرات
▼

1. مقارنة سريعة بين البنى التكرارية

البنية متى تستعمل؟
Tant que … Faire عندما نجهل عدد التكرارات مسبقًا، لكن لدينا شرط توقّف.
Pour … Faire عندما نعرف عدد التكرارات (من i = 1 إلى N مثلًا).
Répéter … Jusqu’à عندما نريد تنفيذ التعليمات على الأقل مرة واحدة ثم نختبر الشرط.

2. مثال: مجموع الأعداد من 1 إلى n (Tant que)

Algorithme SommeN_TantQue
Var n, s, i : Entier
Début
  Écrire "Entrer un entier"
  Lire n
  s <- 0
  i <- 1
  Tant que i <= n Faire
    s <- s + i
    i <- i + 1
  Fin Tant que
  Écrire "La somme est", s
Fin

3. نفس المثال ببنية Pour

Algorithme SommeN_Pour
Var n, s, i : Entier
Début
  Écrire "Entrer un entier"
  Lire n
  s <- 0
  Pour i <- 1 à n Faire
    s <- s + i
  Fin Pour
  Écrire "La somme est", s
Fin

المخططات الانسيابية (Organigrammes)

تمثيل الخوارزمية بالرسم
▼

المخطط الانسيابي هو تمثيل رسومي للخوارزمية باستعمال رموز وأشكال متفق عليها، توضح مسار التنفيذ من البداية إلى النهاية.

1. الرموز الأساسية

الشكل الدور مثال
بيضوي (Oval) البداية والنهاية DEBUT ، FIN
مستطيل (Rectangle) تعليمة معالجة som <- x + y
متوازي أضلاع إدخال/إخراج Lire x أو Écrire som
معيّن (Losange) اختبار شرط x > 0 ?
سهم اتجاه التنفيذ من تعليمة إلى أخرى
🧠
لتتأكد أن المخطط صحيح، اتبع الأسهم كما لو كنت الحاسوب: يجب أن تصل دائمًا من البداية إلى النهاية دون أن تبقى عالقًا في مكان ما.

أمثلة خوارزميات كاملة (من الدرس)

للتدريب على الكتابة المنهجية
▼

1. خوارزمية جمع عددين

Algorithme Addition
Var x, y, som : Entier
Début
  Écrire "Entrer les valeurs de x et y"
  Lire x, y
  som <- x + y
  Écrire "La somme =", som
Fin

2. خوارزمية حساب متوسط ثلاث علامات

Algorithme Moyenne3
Var s1, s2, s3, Moy : Réel
Début
  Écrire "Entrer les trois notes"
  Lire s1, s2, s3
  Moy <- (s1 + s2 + s3) / 3
  Écrire "La moyenne est", Moy
Fin

3. خوارزمية إشارة عدد حقيقي

Algorithme SigneNombre
Var x : Réel
Début
  Écrire "Donnez un nombre"
  Lire x
  Si x < 0 Alors
    Écrire "x est strictement négatif"
  Sinon Si x > 0 Alors
    Écrire "x est strictement positif"
  Sinon
    Écrire "x est nul"
  Fin Si
Fin
✅
حاول أن تعيد كتابة هذه الخوارزميات لوحدك بدون النظر، ثم عد للتحقق. إذا استطعت ذلك فمعناه أنك فهمت المنهجية وليس المثال فقط.
